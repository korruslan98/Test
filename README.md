# Inventory Sync Hub

Сервис для:
- подключения к **МойСклад** и импорта реального каталога товаров;
- хранения каталога на сервере;
- добавления нескольких магазинов **Wildberries** и **Ozon**;
- синхронизации остатков из каталога в маркетплейсы по API.

## Что умеет
- импортировать товары из `МойСклад` через API;
- хранить каталог и токены на сервере в `data/app-state.json`;
- поддерживать несколько магазинов WB и Ozon одновременно;
- синхронизировать остатки по всем подключённым магазинам;
- показывать каталог в виде карточек с миниатюрой, остатком и оптовой ценой;
- открывать полную карточку товара (галерея изображений, описание, характеристики) по клику;
- показывать на товаре значки `WB` и `Ozon` с подсказкой, в какие магазины товар уходит;
- автоматически подставлять `код` товара из МойСклад в:
  - **WB**: `Артикул продавца / SKU`;
  - **Ozon**: `offer_id / Артикул`;
- автоматически запускать синхронизацию остатков каждые **15 минут**;
- автоматически включать галочку маркетплейса в карточке товара, если уже создан токен соответствующего магазина;
- массово включать синхронизацию WB или Ozon для всех товаров одной кнопкой;
- редактировать реальные идентификаторы товара для синхронизации:
  - **WB**: `sku` или `chrtId`;
  - **Ozon**: `offer_id` или `product_id`, при необходимости свой `warehouse_id`.

## Требования
- Node.js **18+**
- npm **9+**

## Запуск
```bash
npm install
npm start
```

После запуска приложение будет доступно по адресу:

```text
http://localhost:3000
```

## Режим разработки
```bash
npm run dev
```

## Как настроить

### 1. Подключить МойСклад
В интерфейсе нажмите **«Подключить МойСклад»** и заполните:
- `Название подключения`;
- `URL API` — обычно `https://api.moysklad.ru/api/remap/1.2`; если вставите ссылку из документации `https://dev.moysklad.ru/doc/api/remap/1.2`, сервис автоматически заменит её на правильный API URL;
- `Режим авторизации`:
  - `Bearer token`, если используете токен;
  - `Логин / пароль`, если используете basic auth.

После сохранения сервер проверит подключение к МойСклад.

### 2. Импортировать товары
Нажмите **«Импортировать товары»** — сервер запросит товары из МойСклад и сохранит их локально.

После импорта:
- `код` товара из МойСклад автоматически записывается в `WB SKU / Артикул продавца`;
- этот же `код` автоматически записывается в `Ozon offer_id / Артикул`;
- оба маркетплейса у импортированного товара включаются автоматически, чтобы остатки могли сразу участвовать в синхронизации.
- если вы добавляете товар вручную, галочка WB/Ozon тоже включается автоматически, когда в системе уже есть токен соответствующего маркетплейса.
- в каталоге можно нажать **«Синхронизировать WB всем»** или **«Синхронизировать Ozon всем»**, чтобы массово поставить галочку для всех товаров.

### Навигация (оглавление слева)
- в интерфейсе основные разделы вынесены в левое оглавление: **Каталог**, **Маркетплейсы**, **Как это работает**;
- в разделе **Каталог** отображаются карточки товаров из МойСклад;
- по нажатию на карточку открывается подробный просмотр товара.

### 3. Добавить магазины
Во вкладке **«Маркетплейсы»** нажмите **«Добавить токен»**.

#### Для Wildberries
Нужно указать:
- название магазина;
- `ID склада`;
- `WB API токен`. Документация WB находится на `https://dev.wildberries.ru`, но реальные запросы по Marketplace API идут на `https://marketplace-api.wildberries.ru`.

#### Для Ozon
Нужно указать:
- название магазина;
- `warehouse_id`;
- `Client ID`;
- `API Key`.

### 4. Настроить идентификаторы товаров
У каждого товара нажмите **«Редактировать»** и задайте:

#### Для WB
- `WB SKU` **или** `WB chrtId`.

#### Для Ozon
- `offer_id` **или** `product_id`.
- при необходимости индивидуальный `warehouse_id` на уровне товара.

### 5. Синхронизировать остатки
Нажмите **«Синхронизировать остатки»**.

Сервер отправит:
- в **Wildberries** — `PUT /api/v3/stocks/{warehouseId}`;
- в **Ozon** — `POST /v2/products/stocks`.

Кроме ручного запуска, сервер автоматически повторяет синхронизацию каждые **15 минут**.

## Где хранятся данные
Состояние приложения сохраняется в файл:

```text
data/app-state.json
```

Этот файл не коммитится в git.

## Важно
- Токены и ключи хранятся на сервере в локальном JSON-файле. Для production лучше заменить это на БД и шифрование секретов.
- Для реальной синхронизации у товаров должны быть корректные идентификаторы WB/Ozon.
- Если API маркетплейса вернёт ошибку, она появится в статусе синхронизации у товара.
